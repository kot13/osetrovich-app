# Quickstart: Обновление каталога по контракту

**Фича**: [spec.md](./spec.md) | **План**: [plan.md](./plan.md) | **Модель**: [data-model.md](./data-model.md)

Руководство по проверке фичи после реализации.

## Предварительные требования

- Flutter SDK (stable), Dart 3.x
- Репозиторий с выполненными задачами из `tasks.md`
- Мок-API включён (`useMockApi = true` в `lib/core/network/providers.dart`)
- OpenAPI и моки содержат `integer` id и поля `sale` / `special`

## Установка и запуск

```bash
flutter pub get
flutter run
```

## Автоматические тесты

```bash
flutter test test/features/catalog/
flutter test test/features/cart/cart_notifier_test.dart
flutter test test/core/network/mock_api_client_products_test.dart
flutter analyze
```

**Ожидание**: все тесты зелёные; widget-тесты `product_card_test` проверяют наличие/отсутствие
бейджей «Акция» и «СПЕЦЦЕНА».

## Сценарии ручной проверки

### С1. Числовые id и категории (US1, FR-001–FR-003)

1. Открыть «Каталог».
2. Убедиться, что Filter Chips загружаются без ошибок.
3. Выбрать «Рыба» — сетка товаров отображается.
4. Нажать на карточку — открывается деталь с корректным названием и ценой.
5. Вернуться, добавить товар в корзину — бейдж корзины обновляется.

### С2. Категория «Все» (Edge case)

1. Выбрать chip «Все».
2. Отображаются товары из всех категорий; ошибок запроса нет.

### С3. Бейдж «Акция» (US2, FR-005)

1. В моке найти товар с `sale: true` (см. `mock_api_client.dart` или категорию с акционными
   позициями).
2. На карточке в сетке виден бейдж **«Акция»** (жёлтый фон).
3. Товар с `sale: false` — бейджа нет.

### С4. Бейдж «СПЕЦЦЕНА» (US3, FR-006)

1. Найти товар с `special: true`.
2. На карточке виден бейдж **«СПЕЦЦЕНА»** (тёмно-синий фон).
3. Товар с `special: false` — бейджа нет.

### С5. Оба бейджа (FR-009)

1. Найти товар с `sale: true` и `special: true`.
2. На карточке отображаются **оба** бейджа слева на фото.

### С6. Товары недели (FR-007)

1. Открыть «Главная», блок «Товары недели».
2. Карточки показывают те же бейджи по правилам С3–С5.
3. Тап по карточке ведёт на деталь товара; корзина работает.

### С7. Боевой API (опционально)

1. `useMockApi = false` в `providers.dart`.
2. `flutter run`.
3. Каталог загружается с `https://trout.osetrovich.ru/v1`; id и бейджи соответствуют
   ответу сервера.

## Критерии готовности

- [ ] Нет runtime-ошибок парсинга JSON (type cast int/string)
- [ ] Корзина и навигация работают с числовыми id
- [ ] Бейджи не перекрывают кнопку цены и название
- [ ] `flutter test` и `flutter analyze` без ошибок
