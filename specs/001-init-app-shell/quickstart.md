# Quickstart: Инициализация приложения

**Фича**: [spec.md](./spec.md) | **План**: [plan.md](./plan.md) | **Модель**: [data-model.md](./data-model.md)

Руководство по проверке фичи после реализации (`/speckit-implement`). Не дублирует код —
только шаги валидации.

## Предварительные требования

- Flutter SDK (stable), Dart 3.x
- Xcode (iOS) или Android Studio / эмулятор (Android)
- Репозиторий с выполненными задачами из `tasks.md`

## Установка и запуск

```bash
# из корня репозитория
flutter pub get
flutter run
```

Для мок-API убедитесь, что в конфигурации включён режим моков (см. `lib/core/network/` —
флаг `useMockApi` или аналог из plan.md).

## Автоматические тесты

```bash
# unit + widget
flutter test

# integration (эмулятор/устройство должно быть подключено)
flutter test integration_test/
```

**Ожидание**: все тесты зелёные; покрыты tab navigation, auth flow, empty states, catalog chips.

## Сценарии ручной проверки

### С0. Сплэш и имя приложения (FR-018, FR-019, SC-006)

1. Полностью закрыть приложение и запустить снова (холодный старт).
2. На сплэше: логотип osetrovich на фоне `#213C57` (не дефолтная иконка Flutter).
3. На домашнем экране устройства имя приложения — «Осетрович».

---

### С1. Tab Bar (US1, FR-001, FR-002)

1. Запустить приложение.
2. Убедиться: 5 вкладок — «Главная», «Каталог», «Акции и новости», «Корзина», «Профиль».
3. Поочерёдно открыть каждую вкладку — активная подсвечена.
4. Открыть «Корзина» → «Профиль» → снова «Корзина» — экран корзины не сбросился.

**Ожидание**: мгновенное переключение, русские подписи.

---

### С2. Пустая корзина (US3, FR-003)

1. Вкладка «Корзина».
2. Текст: «В корзине пока пусто».
3. Нажать кнопку перехода в каталог.

**Ожидание**: открывается вкладка «Каталог».

---

### С3. Профиль без авторизации (US3, FR-004)

1. Вкладка «Профиль» (без входа).
2. Текст: «Необходима авторизация».
3. Нажать кнопку входа.

**Ожидание**: экран ввода телефона.

---

### С4. Авторизация по СМС (US2, FR-005–FR-009)

1. Ввести телефон `+7 (916) 123-45-67`.
2. Подтвердить — открывается экран кода.
3. Проверить: кнопка «Повторить запрос» неактивна, виден отсчёт ~60 с.
4. Нажать «Назад» — возврат к телефону, номер сохранён.
5. Снова запросить код, ввести `123456` (мок).

**Ожидание**: успешный вход; на «Профиле» нет заглушки; на «Главной» скрыт блок «Авторизуйтесь».

**Негатив**: код `000000` → сообщение «Неверный код» на русском.

---

### С5. Главная (US4, FR-010–FR-012)

1. Вкладка «Главная».
2. В шапке справа — колокольчик со счётчиком (мок: 3).
3. Тап по колокольчику — навигации нет (scope).
4. Ниже — карусель баннеров (свайп).
5. Без авторизации — блок «Авторизуйтесь».

---

### С6. Каталог и категории (US5, FR-013–FR-015)

1. После холодного старта открыть «Каталог» ≤ 3 с.
2. Видны Filter Chips: Все, Рыба, Икра, … (12 шт.).
3. По умолчанию активен «Все».
4. Переключить чип — визуальное выделение.
5. Состояние списка: «Ничего не нашлось».

---

### С7. Акции и новости (US3, FR-016)

1. Вкладка «Акции и новости».
2. Текст: «Ничего не нашлось».

---

### С8. Визуальная проверка (конституция VIII)

- Primary `#213C57`, Accent `#FFB400`, Background `#F4F5F5`, текст `#252A2F`.
- Нет хардкода чужих цветов на брендовых элементах.
