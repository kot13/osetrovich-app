# Quickstart: Корзина и оформление заказа

**Фича**: [spec.md](./spec.md) | **План**: [plan.md](./plan.md) | **Модель**: [data-model.md](./data-model.md)

Руководство по проверке фичи после реализации.

## Предварительные требования

- Flutter SDK (stable), Dart 3.x
- Репозиторий с выполненными задачами из `tasks.md`
- Мок-API включён (`useMockApi = true` в `lib/core/network/providers.dart`)
- Для оформления заказа — **авторизованная** сессия (СМС-вход)
- Каталог товаров реализован (`004-product-catalog`)

## Установка и запуск

```bash
flutter pub get
flutter run
```

## Сценарии ручной проверки

### С1. Пустая корзина (FR-010)

1. Открыть вкладку «Корзина» без добавленных товаров.
2. Убедиться: «В корзине пока пусто», кнопка «Перейти в каталог».
3. Нажать кнопку — переход на вкладку «Каталог».

---

### С2. Непустая корзина — состав и итоги (US1, FR-001–FR-003)

1. В каталоге добавить 2 разных товара.
2. Открыть «Корзину».
3. Убедиться: список позиций (название, вес, цена, количество, сумма по строке).
4. Проверить блок итогов: «Товары», «Доставка», «Итого».
5. При сумме товаров **< 2000 ₽** — доставка **300 ₽**.
6. Добавить товары до суммы **≥ 2000 ₽** — доставка **«Бесплатно»** / **0 ₽**.

---

### С3. Изменение количества на экране корзины (US2, FR-002, FR-011)

1. На экране корзины нажать «+» на позиции — сумма строки и итог обновились.
2. Нажать «−» до удаления позиции — строка исчезла; бейдж Tab Bar уменьшился.
3. Удалить последнюю позицию — пустое состояние корзины.

---

### С4. Условия оплаты и доставки (FR-005)

1. С непустой корзиной прокрутить до информационного блока.
2. Текст совпадает со spec (сообщение после заказа, порог 2000 ₽, 300 ₽, оплата при получении).

---

### С5. Валидация адреса (US3, FR-006, FR-008)

1. Авторизоваться (если не в сессии).
2. Оставить адрес пустым, нажать «Оформить».
3. Сообщение «Укажите адрес доставки»; корзина не очищена.

---

### С6. Оформление без авторизации (FR-012, FR-013)

1. Выйти из аккаунта (Профиль → Выйти).
2. Наполнить корзину, открыть «Корзину», **заполнить адрес** (и при желании комментарий).
3. Нажать «Оформить» — открывается экран ввода телефона (клавиатура сразу в фокусе).
4. Пройти авторизацию (телефон → СМС; на экране кода клавиатура сразу в фокусе).
5. После успешного кода — возврат на «Корзину», **автоматическое** оформление заказа.
6. Диалог об успехе; корзина пуста. Повторно нажимать «Оформить» не нужно.

---

### С7. Успешное оформление (US3, FR-007–FR-009, SC-004)

1. Войти в аккаунт.
2. Добавить товары, открыть «Корзину».
3. Заполнить адрес; опционально — комментарий.
4. Нажать «Оформить».
5. Диалог/сообщение об успешном оформлении.
6. После закрытия — пустая корзина, бейдж Tab Bar скрыт.
7. Каталог: карточки товаров в состоянии «не в корзине».

---

### С8. Ошибка сети (Edge case)

1. Симулировать ошибку API (отключить мок / тестовый флаг ошибки, если есть).
2. Нажать «Оформить» — сообщение об ошибке.
3. Корзина и введённые поля сохранены.

---

### С9. Двойное нажатие «Оформить» (Edge case)

1. При медленном моке дважды быстро нажать «Оформить».
2. Создан только один заказ; кнопка заблокирована на время отправки.
