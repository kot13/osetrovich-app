# Спецификация фичи: Каталог товаров

> **Язык**: этот документ и все артефакты Spec Kit MUST быть на **русском языке**
> (конституция проекта, принцип VII).

**Feature Branch**: `004-product-catalog`

**Created**: 2026-07-14

**Status**: Implemented

**Input**: User description: "добавляем каталог. Гридвью из товаров. На телефоне 2 карточки в
ряд. Подгружаются по 20 шт при скроле. Карточка: фото, название (2 строки), вес, кнопка с
ценой и знаком плюс; при добавлении — панель «− [количество] × [цена] +»; реактивно
меняется счётчик в таббаре корзины. Клик по карточке — страница товара с листаемыми фото,
названием, весом, ценой, описанием и закреплённой панелью добавления в корзину."

## User Scenarios & Testing *(mandatory)*

### User Story 1 — Просмотр каталога и подгрузка товаров (Priority: P1)

Покупатель открывает вкладку «Каталог», выбирает категорию (Filter Chips из init-app-shell)
и видит сетку товаров: на экране телефона — **2 карточки в ряд**. При прокрутке вниз
автоматически подгружаются следующие порции по **20 товаров**, пока в категории есть
доступные позиции.

**Why this priority**: без списка товаров каталог не выполняет основную функцию магазина;
это базовый сценарий покупки.

**Independent Test**: открыть «Каталог», выбрать категорию с товарами, убедиться в сетке
2×N, прокрутить до конца первой страницы и проверить появление следующих 20 позиций.

**Acceptance Scenarios**:

1. **Given** пользователь на вкладке «Каталог» и выбрана категория с товарами,
   **When** экран загружается, **Then** отображается сетка из двух колонок с карточками
   товаров; пустое состояние «Ничего не нашлось» не показывается.
2. **Given** отображена первая порция товаров (до 20 шт.), **When** пользователь
   прокручивает список к концу видимой области, **Then** подгружается следующая порция
   до 20 товаров без перезагрузки всего экрана.
3. **Given** в выбранной категории меньше 20 товаров, **When** пользователь открывает
   каталог, **Then** отображаются все доступные товары; индикатор подгрузки не
   появляется повторно после исчерпания списка.
4. **Given** пользователь меняет категорию в Filter Chips, **When** выбрана другая
   категория, **Then** сетка обновляется и показывает товары новой категории с начала
   списка (сброс позиции прокрутки).
5. **Given** в выбранной категории нет товаров, **When** пользователь смотрит на экран,
   **Then** отображается состояние «Ничего не нашлось» (как в init-app-shell).

### User Story 2 — Быстрое добавление в корзину с карточки (Priority: P1)

Покупатель добавляет товар в корзину прямо из сетки, не открывая детальную страницу.
Карточка показывает фото, название (до двух строк с обрезкой), вес и кнопку с ценой и
знаком «+». После первого добавления кнопка меняется на компактную панель
«− [количество] × [цена] +»; «−» и «+» изменяют количество. Фото, название и вес
остаются видимыми. Счётчик на вкладке «Корзина» в нижнем Tab Bar **реактивно** отражает
число **разных позиций** (уникальных товаров) в корзине.

**Why this priority**: быстрое добавление — ключевой e-commerce паттерн; связь с корзиной
даёт пользователю обратную связь без перехода на другую вкладку.

**Independent Test**: добавить два разных товара из сетки, проверить визуальное состояние
карточек и значение «2» на бейдже корзины; уменьшить один товар до нуля и убедиться, что
бейдж стал «1», а карточка вернулась в исходный вид.

**Acceptance Scenarios**:

1. **Given** товар ещё не в корзине, **When** пользователь нажимает кнопку «[цена] +»,
   **Then** количество становится 1; кнопка показывает «− 1 × [цена] +»; фото, название
   и вес остаются на месте.
2. **Given** товар уже в корзине (количество ≥ 1), **When** пользователь нажимает «+» на
   карточке, **Then** количество увеличивается на 1; панель «− K × [цена] +» обновляется
   без задержки, заметной пользователю.
3. **Given** количество товара больше 1, **When** пользователь нажимает «−»,
   **Then** количество уменьшается на 1.
4. **Given** количество товара равно 1, **When** пользователь нажимает «−»,
   **Then** товар удаляется из корзины; карточка возвращается к виду с кнопкой «[цена] +».
5. **Given** в корзине N различных товаров (N ≥ 0), **When** пользователь добавляет или
   удаляет позиции с карточек, **Then** бейдж на вкладке «Корзина» показывает N без
   необходимости переключаться на вкладку.
6. **Given** пользователь увеличивает количество уже добавленного товара,
   **When** счётчик на Tab Bar обновляется, **Then** число **разных** позиций не
   меняется (меняется только количество внутри одной позиции).

### User Story 3 — Страница товара и добавление из деталей (Priority: P2)

Покупатель нажимает на карточку (не на кнопку цены) и переходит на страницу товара.
Видит фото (при нескольких — горизонтальная прокрутка/листание), название, вес, цену и
описание. Внизу экрана — закреплённая панель добавления в корзину (в составе `body`,
не `bottomNavigationBar` — совместимость с Tab Bar): если товар не добавлен — компактная
кнопка «[цена] +» слева; если добавлен — «− [количество] × [цена] +» на всю ширину.

**Why this priority**: детальная страница нужна для осознанной покупки и дополнительной
информации; вторична относительно быстрого просмотра сетки.

**Independent Test**: открыть товар из сетки, проверить все поля; добавить через floating-
кнопку; вернуться назад и убедиться, что карточка в сетке и бейдж корзины синхронизированы.

**Acceptance Scenarios**:

1. **Given** пользователь на экране каталога, **When** он нажимает на область карточки
   (фото, название, вес), **Then** открывается страница товара с актуальными данными
   выбранной позиции.
2. **Given** у товара одно фото, **When** открыта страница товара, **Then** отображается
   это фото без индикаторов листания.
3. **Given** у товара несколько фото, **When** пользователь листает галерею горизонтально,
   **Then** поочерёдно показываются все изображения товара.
4. **Given** товар не в корзине, **When** открыта страница товара, **Then** floating-
   панель показывает кнопку «[цена] +».
5. **Given** товар в корзине с количеством K, **When** открыта страница товара,
   **Then** floating-панель показывает «− K × [цена] +»; «−» и «+» изменяют K по тем
   же правилам, что и на карточке в сетке.
6. **Given** пользователь изменил количество на странице товара, **When** он возвращается
   к сетке каталога, **Then** соответствующая карточка отражает то же количество и
   визуальное состояние панели «− K × [цена] +».
7. **Given** пользователь на странице товара, **When** он нажимает системную кнопку
   «Назад», **Then** он возвращается к сетке каталога с сохранением выбранной категории
   и позиции прокрутки.

### Edge Cases

- Ошибка загрузки первой страницы товаров: понятное сообщение на русском и действие
  «Повторить» (аналогично загрузке категорий).
- Ошибка подгрузки следующей страницы: пользователь может повторить попытку; уже
  загруженные товары остаются на экране.
- Медленная сеть: при первой загрузке и при подгрузке показывается индикатор загрузки.
- Недоступное или битое изображение: placeholder вместо фото на карточке и на странице
  товара.
- Очень длинное название: на карточке обрезается до двух строк с многоточием; на странице
  товара название показывается полностью (с переносом строк).
- Быстрые повторные нажатия «+»/«−»: количество не «уезжает» — каждое нажатие учитывается
  ровно один раз.
- Нажатие на кнопку «+»/«−» на карточке **не** открывает страницу товара (событие не
  всплывает на переход).
- Одновременное изменение количества на странице товара и в сетке: везде отображается
  одно актуальное значение.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Экран «Каталог» MUST отображать товары выбранной категории в виде сетки с
  **двумя колонками** на типовом экране телефона.
- **FR-002**: Система MUST загружать товары постранично: **20 позиций** за один запрос;
  следующая страница MUST запрашиваться при достижении пользователем конца прокручиваемого
  списка.
- **FR-003**: Каждая карточка товара в сетке MUST содержать: фото, название (максимум
  **2 строки** с обрезкой), вес, кнопку с ценой и «+».
- **FR-004**: При добавлении товара с карточки MUST изменяться вид кнопки на компактную
  панель «− [количество] × [цена] +»; фото, название (2 строки) и вес MUST оставаться
  видимыми (без тени и без наложения количества по центру карточки).
- **FR-005**: Кнопки «−» и «+» на карточке MUST увеличивать и уменьшать количество товара
  в корзине; при уменьшении до нуля товар MUST удаляться из корзины, карточка MUST
  возвращаться к исходному виду.
- **FR-006**: Tab Bar MUST показывать на вкладке «Корзина» **реактивный** счётчик числа
  **различных товарных позиций** (уникальных SKU) в корзине; при нуле позиций бейдж MUST
  быть скрыт или показывать «0» согласно единому паттерну приложения.
- **FR-007**: Нажатие на карточку (кроме области управления количеством) MUST открывать
  **страницу товара** в рамках навигационного стека вкладки «Каталог».
- **FR-008**: Страница товара MUST отображать: фото (с горизонтальным листанием при
  нескольких изображениях), название, вес, цену, текстовое описание.
- **FR-009**: Страница товара MUST иметь закреплённую внизу панель добавления (в `Column`
  внутри `body`, над Tab Bar): компактная «[цена] +» слева, если товар не в корзине;
  «− [количество] × [цена] +» на всю ширину, если товар в корзине. Контент (галерея,
  название, описание) MUST отображаться над панелью.
- **FR-010**: Изменение количества на странице товара MUST синхронизироваться с карточкой
  в сетке и со счётчиком вкладки «Корзина» в реальном времени.
- **FR-011**: Смена категории MUST сбрасывать список товаров и загружать первую страницу
  для новой категории.
- **FR-012**: Цена MUST отображаться в привычном для магазина формате на русском языке
  (например, «300 ₽»); знак валюты и пробелы — единообразно во всём каталоге.
- **FR-013**: Состояния загрузки, пустого списка и ошибки MUST сопровождаться текстами на
  **русском языке**.
- **FR-014**: Добавление в корзину MUST быть доступно **без обязательной авторизации**
  (просмотр каталога и наполнение корзины — до оформления заказа в отдельной фиче).

### Key Entities

- **Товар (Product)**: позиция каталога — идентификатор, название, вес (отображаемая
  строка, например «500 г»), цена за единицу, одно или несколько изображений, описание,
  принадлежность к категории.
- **Позиция корзины (Cart Line Item)**: связь товара с количеством в корзине пользователя;
  уникальность по товару (одна строка на SKU).
- **Корзина (Cart)**: совокупность позиций корзины; агрегированный показатель для Tab Bar —
  **число различных позиций** (не суммарное количество штук).
- **Страница каталога (Catalog Page)**: порция списка товаров категории (до 20 записей) с
  метаданными для подгрузки следующей страницы.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Пользователь видит первые товары категории не более чем через **3 секунды**
  при типичном мобильном соединении.
- **SC-002**: Подгрузка следующих 20 товаров воспринимается как непрерывный скролл:
  **95%** попыток подгрузки завершаются до того, как пользователь дойдёт до «пустого» места
  в конце списка (при наличии следующей страницы).
- **SC-003**: Добавление товара с карточки и обновление бейджа корзины происходят **без
  видимой задержки** — пользователь видит новое состояние сразу после нажатия.
- **SC-004**: **100%** сценариев из User Stories 1–3 проходят в автоматизированных
  widget/integration-тестах основных потоков (согласно конституции, принцип III).
- **SC-005**: Не менее **90%** участников usability-проверки (или команды) без подсказок
  успешно добавляют товар из сетки, меняют количество и открывают страницу товара с первой
  попытки.

## Assumptions

- Фича **расширяет** экран «Каталог» из `001-init-app-shell`: Filter Chips категорий,
  AppBar и вкладка Tab Bar сохраняются; пустое состояние «Ничего не нашлось» переиспользуется.
- Полноценный экран «Корзина» (список позиций, итог, оформление) **вне scope** этой фичи;
  здесь реализуются наполнение корзины, синхронизация состояния и бейдж на Tab Bar.
- Данные товаров и пагинация MUST быть описаны в OpenAPI и доступны через моки (конституция,
  принцип VI); категории уже есть в контракте init-app-shell.
- Вес товара приходит готовой строкой для отображения (не конвертация единиц на клиенте).
- На планшетах и широких экранах число колонок MAY увеличиваться в plan.md; минимальное
  требование спеки — **2 колонки на телефоне**.
- Бейдж корзины считает **уникальные SKU**, а не сумму штук (как указано в описании фичи).
- Корзина хранится локально в сессии приложения до реализации синхронизации с сервером в
  последующих фичах.
- Авторизация для просмотра каталога и добавления в корзину не требуется.
