# Спецификация фичи: Блок статуса лояльности на главной

> **Язык**: этот документ и все артефакты Spec Kit MUST быть на **русском языке**
> (конституция проекта, принцип VII).

**Feature Branch**: `014-home-loyalty-status`

**Created**: 2026-07-21

**Status**: Implemented

**Input**: User description: "добавляем блок со статусом в системе лояльности на главный экран.
1) Данные приходят в профиле (обновлен openapi)
2) На главном экране убираем кнопку Связаться. Вместо неё ставим кнопку Авторизоваться
3) Если пользователь авторизован и у него есть статус - выводим блок с этим статусом.
- Название статуса
- Скидка (если 0 — прячем строку скидки, не весь блок)
- Номер карты (может быть пустым)
4) Возможные статусы: Super VIP, VIP, Elite, Premium, Друг, Участник клуба
5) Скидка лояльности применяется в корзине (кроме акционных товаров; для VIP/Super VIP — на все)"

## User Scenarios & Testing *(mandatory)*

### User Story 1 — Призыв к авторизации вместо «Связаться» (Priority: P1)

Гость на вкладке «Главная» больше не видит кнопку «Связаться» в зоне под баннерами.
Вместо неё отображается кнопка «Авторизоваться», ведущая в сценарий входа по номеру
телефона. Связь с магазином остаётся доступной в профиле и в блоке текущего заказа
(«Связаться с оператором»).

**Why this priority**: замена кнопки — базовое изменение главной; без неё нельзя корректно
разместить блок лояльности и не дублировать призывы к входу.

**Independent Test**: открыть «Главную» без авторизации, убедиться, что кнопки
«Связаться» нет, есть «Авторизоваться»; нажатие открывает экран входа.

**Acceptance Scenarios**:

1. **Given** пользователь не авторизован и находится на «Главной», **When** экран
   отображён, **Then** кнопка «Связаться» в зоне под баннерами отсутствует.
2. **Given** пользователь не авторизован, **When** он видит «Главную», **Then** под
   баннерами отображается кнопка «Авторизоваться».
3. **Given** пользователь нажимает «Авторизоваться», **When** действие обработано,
   **Then** открывается сценарий входа по телефону.
4. **Given** пользователь не авторизован, **When** он просматривает «Главную», **Then**
   дублирующий баннер «Авторизуйтесь» внизу экрана не отображается (единственный призыв
   к входу — кнопка «Авторизоваться» под баннерами).

---

### User Story 2 — Блок статуса лояльности для авторизованного покупателя (Priority: P1)

Авторизованный покупатель с назначенным статусом в программе лояльности видит на «Главной»
(на месте бывшей кнопки «Связаться») информационный блок: название статуса, персональную
скидку (если больше нуля) и номер клубной карты (если указан в профиле). Данные
подгружаются из профиля пользователя при открытии «Главной».

**Why this priority**: основная ценность фичи — показать покупателю его уровень и выгоду
сразу на главной, без перехода в профиль.

**Independent Test**: войти под учётной записью со статусом лояльности, открыть «Главную»,
проверить отображение статуса, скидки и номера карты согласно данным профиля.

**Acceptance Scenarios**:

1. **Given** пользователь авторизован и в профиле указан статус лояльности, **When**
   «Главная» загружена, **Then** под баннерами отображается блок статуса лояльности.
2. **Given** блок статуса отображён, **When** пользователь читает его, **Then** видит
   название статуса на русском языке из фиксированного набора: «Super VIP», «VIP»,
   «Elite», «Premium», «Друг», «Участник клуба».
3. **Given** в профиле скидка больше нуля, **When** блок статуса отображён, **Then**
   в блоке видна персональная скидка в процентах (например, «Скидка 10%»).
4. **Given** в профиле скидка равна нулю, **When** блок статуса отображён, **Then**
   строка со скидкой в блоке не показывается; название статуса и номер карты (если есть)
   остаются видимыми.
5. **Given** в профиле указан номер карты, **When** блок статуса отображён, **Then**
   номер карты отображается в блоке.
6. **Given** в профиле номер карты не указан, **When** блок статуса отображён, **Then**
   строка с номером карты в блоке не показывается.
7. **Given** пользователь авторизован, но статус лояльности в профиле не назначен,
   **When** «Главная» загружена, **Then** блок статуса и кнопка «Авторизоваться» в этой
   зоне не отображаются.
8. **Given** пользователь авторизован с назначенным статусом, **When** он выполняет
   обновление «Главной» (pull-to-refresh), **Then** данные статуса, скидки и карты
   обновляются из профиля.
9. **Given** блок статуса отображён для VIP или Super VIP, **When** пользователь читает
   карточку, **Then** видит бейдж «Максимальный уровень» и условие скидки «на все покупки».
10. **Given** блок статуса отображён для статуса ниже VIP, **When** скидка больше нуля,
    **Then** условие скидки — «кроме товаров по акции».
11. **Given** в профиле указан номер карты, **When** пользователь нажимает кнопку
    копирования или область номера карты, **Then** номер копируется в буфер обмена и
    показывается snackbar «Номер карты скопирован» (тот же стиль, что при успешной оценке
    заказа).

### User Story 4 — Персональная скидка в корзине (Priority: P1)

Авторизованный покупатель с персональной скидкой в профиле видит в корзине сумму товаров
до и после скидки. Скидка рассчитывается от процента в профиле; товары с флагом акции
(`sale: true`) исключаются из расчёта, **кроме** статусов Super VIP и VIP (для них скидка
на все позиции).

**Why this priority**: скидка из профиля должна влиять на итог заказа, иначе блок на главной
не отражает реальную выгоду.

**Independent Test**: войти под `+79001111111` (Premium, 10 %), добавить обычный и акционный
товар — скидка только на обычный; войти под VIP — скидка на оба.

**Acceptance Scenarios**:

1. **Given** пользователь авторизован, скидка в профиле > 0, корзина не пуста, **When**
   открыта корзина, **Then** в итогах видны строки «Товары», «Скидка N%» и «Товары со
   скидкой» (если скидка применена хотя бы к одной позиции).
2. **Given** статус Premium (или ниже VIP) и в корзине есть акционный товар, **When**
   рассчитывается итог, **Then** скидка не применяется к акционным позициям.
3. **Given** статус VIP или Super VIP, **When** рассчитывается итог, **Then** скидка
   применяется ко всем позициям, включая акционные.
4. **Given** скидка в профиле 0 %, **When** открыта корзина, **Then** строки лояльной
   скидки в итогах не отображаются.

---

### User Story 3 — Корректное поведение при смене сессии и ошибках загрузки (Priority: P2)

При входе, выходе или смене аккаунта блок лояльности и кнопка «Авторизоваться» на «Главной»
реактивно соответствуют текущей сессии. При временной ошибке загрузки профиля покупатель
не видит устаревший статус другого пользователя.

**Why this priority**: защита от некорректного отображения персональных данных важна для
доверия, но вторична относительно основного happy path.

**Independent Test**: войти → увидеть блок → выйти → увидеть «Авторизоваться»; войти под
другим номером → увидеть статус нового аккаунта.

**Acceptance Scenarios**:

1. **Given** пользователь вышел из аккаунта на «Главной», **When** экран обновлён,
   **Then** блок лояльности исчезает и появляется кнопка «Авторизоваться».
2. **Given** пользователь вошёл в аккаунт со статусом лояльности, **When** «Главная»
   отображена, **Then** кнопка «Авторизоваться» заменяется блоком лояльности.
3. **Given** профиль не удалось загрузить, **When** пользователь авторизован,
   **Then** блок лояльности не показывается с данными предыдущей сессии; доступно
   повторить загрузку через обновление «Главной».

---

### Edge Cases

- Что если статус назначен, скидка 0, карта пустая? — Показывается только название статуса.
- Что если статус `null`, но скидка > 0? — Блок лояльности не показывается (условие
  отображения — наличие статуса, не скидки).
- Что если пользователь авторизован, но профиль ещё загружается? — Блок лояльности не
  показывается до получения данных; не отображается «Авторизоваться».
- Что если после входа статус изменился на сервере? — После pull-to-refresh или повторного
  открытия «Главной» отображаются актуальные данные профиля.
- Где остаётся «Связаться»? — В профиле (ContactBlock) и в блоке текущего заказа
  («Связаться с оператором»); с главной кнопка «Связаться» убирается.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: На «Главной» MUST NOT отображаться кнопка «Связаться» в зоне под баннерами
  (бывший `HomeContactButton`).
- **FR-002**: Для неавторизованного пользователя в этой зоне MUST отображаться кнопка
  «Авторизоваться», открывающая сценарий входа по телефону.
- **FR-003**: Для неавторизованного пользователя MUST NOT отображаться нижний баннер
  «Авторизуйтесь» на «Главной» (избежание дублирования призыва к входу).
- **FR-004**: Для авторизованного пользователя на «Главной» MUST запрашиваться профиль
  пользователя, содержащий поля статуса лояльности, скидки и номера карты.
- **FR-005**: Блок статуса лояльности MUST отображаться только если пользователь
  авторизован **и** в профиле назначен статус лояльности (не `null`).
- **FR-006**: Блок статуса MUST показывать название статуса из фиксированного набора
  отображаемых значений: «Super VIP», «VIP», «Elite», «Premium», «Друг», «Участник клуба».
- **FR-007**: Строка со скидкой в блоке MUST отображаться только если персональная скидка
  в профиле больше нуля; формат — процент (целое число от 1 до 100).
- **FR-008**: Строка с номером карты в блоке MUST отображаться только если номер карты
  в профиле указан (не пустой).
- **FR-009**: Если пользователь авторизован, но статус лояльности не назначен, зона под
  баннерами MUST оставаться пустой (без кнопки «Авторизоваться» и без блока лояльности).
- **FR-010**: При обновлении «Главной» (pull-to-refresh) для авторизованного пользователя
  MUST обновляться данные профиля, влияющие на блок лояльности.
- **FR-011**: При смене сессии (вход, выход, смена аккаунта) отображение зоны под
  баннерами MUST реактивно соответствовать текущей сессии и профилю.
- **FR-012**: Все пользовательские тексты блока лояльности и кнопки «Авторизоваться» MUST
  быть на русском языке.
- **FR-013**: Визуальное оформление блока лояльности и кнопки «Авторизоваться» MUST
  соответствовать фирменной палитре osetrovich.ru (конституция, принцип VIII).
- **FR-014**: Блок лояльности MUST быть тёмной карточкой (`AppColors.primary`) с двумя
  колонками: слева — иконка медали, заголовок «Ваш статус», название статуса и опциональный
  бейдж; справа (узкая колонка) — блок «Ваша скидка» при `discount > 0`.
- **FR-015**: Для статусов Super VIP и VIP MUST отображаться бейдж «Максимальный уровень»
  в одну строку.
- **FR-016**: Условие скидки в блоке: VIP/Super VIP — «на все покупки»; остальные статусы
  при ненулевой скидке — «кроме товаров по акции».
- **FR-017**: Номер карты MUST отображаться в отдельной нижней полосе с иконкой карты,
  группировкой цифр по 4 и кнопкой копирования; после копирования — snackbar
  «Номер карты скопирован».
- **FR-018**: В корзине MUST рассчитываться персональная скидка из `profile.discount` с
  учётом статуса и флага `sale` у позиций (см. User Story 4).
- **FR-019**: В итогах корзины при применённой скидке MUST отображаться сумма товаров до
  скидки, процент и сумма со скидкой.

### Key Entities

- **Профиль пользователя (расширение)**: помимо существующих полей содержит статус
  лояльности (одно из шести значений или отсутствие), персональную скидку в процентах
  (0–100) и номер клубной/дисконтной карты (опционально).
- **Статус лояльности**: уровень покупателя в программе лояльности; шесть уровней с
  фиксированными отображаемыми названиями (см. FR-006).
- **Блок статуса на главной**: информационная карточка под баннерами; тёмный фон, две
  колонки (статус / скидка), опциональная полоса номера карты; состав зависит от профиля.
- **Скидка в корзине**: расчёт `OrderTotals` с полями `itemsSubtotalBeforeDiscountRub`,
  `loyaltyDiscountRub`, `loyaltyDiscountPercent`; правила eligibility по `sale` и статусу.

### Соответствие уровней статуса

| Значение в данных профиля | Отображаемое название |
|---------------------------|----------------------|
| super_vip | Super VIP |
| vip | VIP |
| elite | Elite |
| premium | Premium |
| friend | Друг |
| club_member | Участник клуба |

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Неавторизованный пользователь видит кнопку «Авторизоваться» вместо
  «Связаться» на «Главной» сразу после открытия экрана.
- **SC-002**: Авторизованный пользователь со статусом лояльности видит блок со статусом
  в течение 3 секунд после открытия «Главной» при стабильном соединении.
- **SC-003**: 100% отображаемых названий статусов соответствуют таблице соответствия
  (FR-006) для всех шести уровней.
- **SC-004**: При скидке 0 % строка скидки не отображается ни в одном сценарии
  тестирования; при скидке > 0 % — отображается всегда.
- **SC-005**: После выхода из аккаунта блок лояльности исчезает в течение 1 секунды без
  перезапуска приложения.
- **SC-006**: 100% видимых пользовательских строк в scope фичи — на русском языке.
- **SC-007**: Скидка в корзине совпадает с процентом профиля и правилами исключения
  акционных товаров для статусов ниже VIP.

## Изменения после реализации (2026-07-24)

- **Скелетоны загрузки**: при первой загрузке профиля на «Главной» показываются shimmer-
  скелетоны блока лояльности и блока лимонов; при pull-to-refresh контент остаётся на
  экране без моргания (`copyWithPrevious` в `ProfileNotifier.refresh`, повторная загрузка
  не сбрасывает данные в пустое состояние).
- **Отступы**: если блок статуса лояльности скрыт (нет статуса), между баннерами и блоком
  лимонов добавлен верхний отступ 16 px.
- **Порядок блоков** уточнён: баннеры → (авторизация / лояльность / пусто) → лимоны →
  «Товары недели» → история заказов.

## Assumptions

- Данные статуса, скидки и номера карты уже включены в ответ профиля пользователя
  (контракт обновлён); отдельный запрос только для главной не требуется.
- Условие показа блока — наличие **статуса лояльности**, а не ненулевой скидки; скидка 0
  скрывает только строку скидки внутри блока, не весь блок.
- Порядок блоков на «Главной» сверху вниз: шапка → баннеры → (кнопка «Авторизоваться»
  **или** блок лояльности **или** пусто) → блок геймификации «лимоны» (для авторизованных) →
  «Товары недели» → «История заказов» (для авторизованных с текущим заказом).
- Кнопка «Связаться» в профиле и «Связаться с оператором» в блоке заказа не затрагиваются
  этой фичей.
- Блок лояльности информационный: нажатие на карточку не ведёт на отдельный экран программы
  лояльности; копирование номера карты — единственное действие в блоке.
- Текст кнопки для гостей — «Авторизоваться» (не «Авторизуйтесь»).
- Зависимости: существующие фичи авторизации (`001-init-app-shell`), профиля
  (`003-user-profile`) и наполнения главной (`007-home-screen-content`).
