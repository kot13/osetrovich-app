# Спецификация фичи: Акции и новости

> (конституция проекта, принцип VII).

**Feature Branch**: `006-promotions-news`

**Created**: 2026-07-14

**Status**: Implemented

**Input**: User description: "наполняем раздел Акции и Новости. На странице: 1) Filter Chips
«Все» / «Акции» / «Новости» (по умолчанию «Все») 2) Лента акций и новостей — карточки с фото,
меткой типа, заголовком и датой в одну колонку. По клику на
карточку открывается страница акции/новости 3) Страница акции или новости содержит: Кнопка

безопасных html-тегов и эмодзи)"

## User Scenarios & Testing *(mandatory)*

### User Story 1 — Фильтрация ленты по типу контента (Priority: P1)

Покупатель открывает вкладку «Акции» и видит горизонтальный ряд Filter Chips с тремя
вариантами: «Все», «Акции» и «Новости». По умолчанию активен чип «Все»; при переключении
лента показывает все материалы или только выбранный тип.

**Independent Test**: открыть вкладку «Акции», убедиться в наличии трёх чипов и корректной
смене содержимого ленты при переключении между «Все», «Акции» и «Новости».

**Acceptance Scenarios**:

1. **Given** пользователь на вкладке «Акции», **When** экран загружается,
   **Then** отображаются три Filter Chip: «Все», «Акции» и «Новости»; активен чип «Все».
2. **Given** активен чип «Все», **When** пользователь нажимает чип «Акции»,
   **Then** чип «Акции» становится активным; лента обновляется и показывает только акции.
3. **Given** активен чип «Акции», **When** пользователь нажимает чип «Новости»,
   **Then** чип «Новости» становится активным; лента обновляется и показывает только новости.
4. **Given** активен чип «Новости», **When** пользователь нажимает чип «Все»,
   **Then** лента обновляется и показывает акции и новости вместе.
5. **Given** для выбранного типа нет опубликованных материалов, **When** пользователь
   смотрит на ленту, **Then** отображается состояние «Ничего не нашлось»; Filter Chips
   остаются доступными для переключения типа.

### User Story 2 — Просмотр ленты акций и новостей (Priority: P1)

Покупатель просматривает вертикальную ленту карточек в **одну колонку**. Каждая карточка
содержит обложечное фото, метку типа («Акция» или «Новость»), заголовок и дату публикации.
Материалы отсортированы от новых к старым. Нажатие на карточку открывает детальную страницу соответствующего материала.

**Why this priority**: лента — основной способ знакомства с акциями и новостями магазина;
без неё раздел остаётся пустой заглушкой.

**Independent Test**: при наличии тестовых данных открыть вкладку «Акции», проверить
отображение карточек в одну колонку и переход на детальную страницу по нажатию.

**Acceptance Scenarios**:

1. **Given** для активного фильтра есть опубликованные материалы, **When** пользователь
   открывает вкладку «Акции», **Then** под Filter Chips отображается вертикальный список
   карточек в одну колонку (по одной карточке на всю ширину контентной области).
2. **Given** отображается лента, **When** пользователь смотрит на карточку,
   **Then** на карточке видны: обложечное фото, метка типа («Акция» или «Новость»),
   заголовок и дата публикации в читаемом формате (например, «14 июля 2026»).
3. **Given** отображается лента из нескольких материалов, **When** пользователь
   просматривает порядок карточек, **Then** более свежие материалы расположены выше
   более старых.
4. **Given** пользователь на экране ленты, **When** он нажимает на карточку,
   **Then** открывается детальная страница выбранного материала.
5. **Given** обложечное фото материала временно недоступно, **When** карточка
   отображается в ленте, **Then** показывается заглушка изображения как в карточке товара
   каталога (нейтральный фон и иконка) без нарушения вёрстки остальных элементов карточки.

### User Story 3 — Чтение детальной страницы акции или новости (Priority: P2)

Покупатель открывает материал из ленты и читает полный текст. На странице отображаются:
кнопка «Назад» в шапке, обложечное фото, метка типа («Акция» или «Новость»), заголовок,
дата публикации и основной текст с поддержкой безопасной разметки и эмодзи.

**Why this priority**: детальная страница раскрывает полное содержание; вторична относительно
просмотра ленты, но необходима для информирования о условиях акций и полных новостях.

**Independent Test**: открыть любой материал из ленты, проверить все поля детальной
страницы и возврат к ленте через кнопку «Назад».

**Acceptance Scenarios**:

1. **Given** пользователь нажал карточку в ленте, **When** открывается детальная
   страница, **Then** в шапке (AppBar в фирменном стиле, как на экране авторизации)
   отображается кнопка «Назад» со стрелкой для возврата к ленте.
2. **Given** открыта детальная страница, **When** пользователь просматривает контент,
   **Then** последовательно отображаются: обложечное фото, метка типа («Акция» или
   «Новость»), заголовок, дата публикации и основной текст.
3. **Given** основной текст содержит разрешённую разметку (абзацы, жирный/курсивный текст,
   списки, ссылки), **When** страница отображается, **Then** разметка визуально
   применяется; эмодзи в тексте отображаются корректно.
4. **Given** основной текст содержит небезопасные или неразрешённые элементы разметки,
   **When** страница отображается, **Then** опасное содержимое не выполняется и не
   отображается; пользователь видит только безопасную часть текста.
5. **Given** пользователь на детальной странице, **When** он нажимает «Назад» в шапке,
   **Then** он возвращается к ленте с сохранением ранее выбранного Filter Chip.
6. **Given** пользователь на детальной странице, **When** он использует системную
   навигацию «назад» (жест или кнопка ОС), **Then** поведение совпадает с кнопкой
   «Назад» в шапке — возврат к ленте.

### Edge Cases

- Что происходит при ошибке загрузки ленты: показывается понятное сообщение об ошибке с
  возможностью повторить попытку; Filter Chips остаются на экране.
- Что происходит при ошибке загрузки детальной страницы: сообщение об ошибке и возможность
  вернуться к ленте.
- Материал удалён или снят с публикации после отображения в ленте: при открытии детальной
  страницы пользователь видит сообщение, что материал недоступен, и может вернуться назад.
- Очень длинный заголовок на карточке: обрезается с многоточием, не ломая вёрстку
  одноколоночного списка.
- Очень длинный основной текст на детальной странице: контент прокручивается вертикально;
  шапка с кнопкой «Назад» остаётся доступной.
- Пустая лента для обоих типов при первом открытии: состояние «Ничего не нашлось» (как в
  `001-init-app-shell`); Filter Chips доступны.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Экран вкладки «Акции» MUST отображать заголовок «Акции» (согласовано с
  `002-notifications-home`).
- **FR-002**: Экран MUST содержать горизонтальный ряд из трёх Filter Chips: «Все»,
  «Акции» и «Новости»; в каждый момент времени активен ровно один чип.
- **FR-003**: При первом открытии экрана MUST быть активен чип «Все».
- **FR-004**: При смене активного чипа лента MUST показывать материалы выбранного фильтра:
  все опубликованные, только акции или только новости.
- **FR-005**: Лента MUST отображать карточки в одну колонку на всю ширину контентной
  области экрана.
- **FR-006**: Каждая карточка в ленте MUST содержать: обложечное фото, метку типа
  («Акция» или «Новость»), заголовок и дату публикации.
- **FR-007**: Материалы в ленте MUST быть отсортированы по дате публикации от новых к
  старым.
- **FR-008**: Нажатие на карточку MUST открывать детальную страницу выбранного материала.
- **FR-009**: Детальная страница MUST содержать в шапке (AppBar, как на экранах
  авторизации) кнопку «Назад» со стрелкой для возврата к ленте.
- **FR-010**: Детальная страница MUST отображать: обложечное фото, метку типа
  («Акция» или «Новость»), заголовок, дату публикации и основной текст.
- **FR-011**: Основной текст MUST поддерживать безопасное отображение разметки: абзацы,
  выделение жирным и курсивом, маркированные и нумерованные списки, гиперссылки; MUST
  поддерживать отображение эмодзи.
- **FR-012**: Система MUST отфильтровывать небезопасную разметку и скрипты; пользователь
  MUST NOT видеть исполняемый или опасный контент в тексте материала.
- **FR-013**: При отсутствии материалов для выбранного типа MUST отображаться состояние
  «Ничего не нашлось»; Filter Chips MUST оставаться интерактивными.
- **FR-014**: При возврате с детальной страницы MUST сохраняться ранее выбранный Filter
  Chip и позиция прокрутки ленты в рамках текущей сессии приложения.
- **FR-015**: Просмотр ленты и детальных страниц MUST быть доступен без авторизации
  (публичный контент).
- **FR-016**: При недоступности обложечного фото MUST отображаться заглушка изображения
  как в карточке товара каталога на карточке ленты и на детальной странице.

### Key Entities

- **Материал (акция/новость)**: единица контента раздела; атрибуты: уникальный
  идентификатор, тип (`акция` | `новость`), заголовок, дата публикации, URL или
  идентификатор обложечного фото, основной текст с разметкой, признак публикации.
- **Тип материала**: дискретное значение для фильтрации и метки на карточке и детальной
  странице — «Акция» или «Новость».
- **Лента**: упорядоченный список опубликованных материалов (всех или одного типа) для
  отображения в одноколоночном списке карточек.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Пользователь может переключиться между фильтрами «Все», «Акции» и «Новости»
  и увидеть обновлённую ленту менее чем за 2 секунды при стабильном соединении.
- **SC-002**: 100% карточек в ленте отображаются в одну колонку с обложечным фото,
  меткой типа, заголовком и датой на типовых экранах телефонов iOS и Android.
- **SC-003**: Пользователь может открыть детальную страницу из ленты и вернуться назад
  одним нажатием; выбранный Filter Chip после возврата совпадает с выбранным до перехода.
- **SC-004**: Текст с разрешённой разметкой и эмодзи на детальной странице читаем и
  визуально соответствует замыслу контента в 100% проверенных тестовых материалов.
- **SC-005**: Ни один тестовый материал с внедрённой небезопасной разметкой не приводит к
  выполнению скриптов или отображению опасного контента при просмотре детальной страницы.
- **SC-006**: При отсутствии контента для выбранного типа пользователь видит «Ничего не
  нашлось» без пустого или «сломанного» экрана.

## Assumptions

- Фича заменяет пустое состояние раздела «Акции» из `001-init-app-shell`; заголовок
  вкладки и экрана — «Акции» (не «Акции и новости»), как в `002-notifications-home`.
- Filter Chips визуально согласованы с чипами категорий в каталоге (активный чип
  выделен акцентным цветом бренда).
- Заглушка обложечного фото на карточке и детали совпадает с карточкой товара в каталоге.
- Контент материалов поставляется через REST API; до подключения боевого API используются
  моки по OpenAPI (конституция, принцип VI).
- Пагинация ленты при прокрутке не входит в scope данной фичи; отображается полный список
  опубликованных материалов, возвращаемый API для выбранного типа (объём — типичный для
  розничного магазина, до нескольких десятков позиций).
- Дата публикации отображается в локали пользователя (русский формат).
- Разрешённые теги разметки ограничены типографикой и структурой текста; вложенные
  медиа, таблицы и произвольные стили вне scope — отображаются как безопасный текст или
  опускаются.
- Редактирование, создание и модерация материалов — на стороне админки/бэкенда; в
  приложении только чтение.
- Deep link на конкретный материал — вне scope; навигация только из ленты вкладки «Акции».
