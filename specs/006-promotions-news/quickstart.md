# Quickstart: Акции и новости

**Фича**: [spec.md](./spec.md) | **План**: [plan.md](./plan.md) | **Модель**: [data-model.md](./data-model.md)

Руководство по проверке фичи после реализации.

## Предварительные требования

- Flutter SDK (stable), Dart 3.x
- Репозиторий с выполненными задачами из `tasks.md`
- Мок-API включён (`useMockApi = true` в `lib/core/network/providers.dart`)
- Авторизация **не** требуется для сценариев акций и новостей

## Установка и запуск

```bash
flutter pub get
flutter run
```

## Автоматические тесты

```bash
flutter test
flutter test integration_test/promotions_flow_test.dart -d <deviceId>
```

**Ожидание**: unit/widget-тесты зелёные; integration — chips → list → detail → back с сохранением фильтра.

## Сценарии ручной проверки

### С1. Filter Chips (US1, FR-002–FR-004)

1. Открыть вкладку «Акции».
2. Убедиться: три чипа «Все», «Акции» и «Новости»; активен «Все».
3. Лента показывает акции и новости вместе.
4. Нажать «Акции» — активен второй чип, лента обновилась (только акции).
5. Нажать «Новости» — только новости.
6. Вернуть «Все» — снова общая лента.

---

### С2. Лента карточек (US2, FR-005–FR-008)

1. При активном чипе с данными в моке — одноколоночный список карточек.
2. На карточке: фото, метка «Акция» или «Новость», заголовок, дата в формате «14 июля 2026».
3. Порядок: более свежие материалы выше.
4. Нажать карточку — открывается детальная страница.

---

### С3. Пустое состояние (US1, FR-013)

1. Если в моке для типа нет материалов (или временно очистить мок) — «Ничего не нашлось».
2. Filter Chips остаются кликабельными; можно переключить тип.

---

### С4. Детальная страница (US3, FR-009–FR-011)

1. Открыть материал из ленты.
2. В шапке — кнопка «Назад» (AppBar в фирменном стиле, как на экране авторизации).
3. Контент сверху вниз: фото, метка «Акция» или «Новость», заголовок, дата, текст.
4. В тексте проверить: **жирный** текст, списки, ссылка (открывается во внешнем браузере), эмодзи.
5. «Назад» — возврат к ленте; активный чип тот же, что до перехода.
6. Системный жест «назад» — то же поведение.

---

### С5. Безопасность HTML (FR-012, SC-005)

1. Открыть мок-материал с внедрённым `<script>` (если добавлен в мок для теста).
2. Скрипт **не** выполняется; опасный контент не отображается.
3. Остальной безопасный текст читаем.

---

### С6. Заглушка фото (FR-016)

1. Открыть материал с невалидным `imageUrl` в моке.
2. На карточке и на детали — заглушка изображения как в карточке товара каталога, вёрстка не ломается.

---

### С7. Ошибки (Edge cases)

1. Симулировать ошибку сети (отключить мок / inject error в тестовой сборке) — сообщение на русском и кнопка повтора на ленте; чипы на месте.
2. Открыть несуществующий id (`/promotions/article/unknown`) — сообщение о недоступности материала, можно вернуться назад.
