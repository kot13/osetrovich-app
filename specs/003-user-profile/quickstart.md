# Quickstart: Профиль пользователя

**Фича**: [spec.md](./spec.md) | **План**: [plan.md](./plan.md) | **Модель**: [data-model.md](./data-model.md)

Руководство по проверке фичи после реализации.

## Предварительные требования

- Flutter SDK (stable), Dart 3.x
- Репозиторий с выполненными задачами из `tasks.md`
- Мок-API включён (`useMockApi = true` в `lib/core/network/providers.dart`)
- Для push: эмулятор/устройство Android 13+ или iOS с запросом notification permission

## Установка и запуск

```bash
flutter pub get
flutter run
```

## Автоматические тесты

```bash
flutter test
flutter test integration_test/profile_flow_test.dart -d <deviceId>
```

**Ожидание**: 53 unit/widget-теста; integration — login (СМС `123456`) → profile → logout.

## Сценарии ручной проверки

### С1. Неавторизованный профиль (US1, FR-002)

1. Без входа открыть вкладку «Профиль».
2. Заглушка «Необходима авторизация» + кнопка «Войти».
3. Ниже доступны «Связаться», оферта, иконки VK/OK.

---

### С2. Просмотр профиля (US1, FR-001, SC-001)

1. Войти по СМС (любой номер + код `123456` в моке).
2. Открыть «Профиль» — имя, телефон, email (если есть).
3. Данные появляются без заметной задержки.

---

### С3. Изменение имени (US2, FR-003)

1. На «Профиле» изменить имя и сохранить.
2. Имя обновилось на экране.
3. Перезапуск приложения — сессия сохранена, имя сохранено (бессрочная сессия).

---

### С4. Смена телефона (US2, FR-004, SC-002)

1. Нажать «Телефон» → ввести новый номер `+7 900 123-45-67`.
2. Экран кода → ввести `123456`.
3. Телефон в профиле обновился.
4. Неверный код → ошибка, старый номер сохранён.
5. Номер `+79999999999` (мок) → «номер уже используется».

---

### С5. Email с подтверждением (US3, FR-005–FR-006, SC-003)

1. Нажать «Email» → ввести `user@example.com`.
2. Экран кода → `123456`.
3. Email отображается как «Подтверждён».
4. Некорректный email → ошибка валидации.
5. `taken@example.com` → «Этот email уже используется».

---

### С6. Push-уведомления (US4, FR-009, SC-004)

1. Переключатель push — выкл/вкл.
2. При первом включении — запрос разрешения ОС.
3. Перезапуск приложения — состояние сохранено.

---

### С7. Поддержка, оферта, соцсети (US5, FR-010–FR-012, SC-005)

1. «Связаться» → звонок `+78125645548` (`ListTile`, без trailing-иконки).
2. «Оферта» → https://osetrovich.ru/privacy-policy
3. VK → https://vk.com/osetrovich (иконка `#252A2F`)
4. OK → https://ok.ru/osetrovich (иконка `#252A2F`)

---

### С8. Выход (US1, FR-013, SC-006)

1. Нажать «Выйти» (кнопка `#FFB400`).
2. Заглушка авторизации.
3. «Главная» / уведомления — без персональных данных сессии.

---

### С9. Бессрочная сессия (FR-008, SC-007)

1. Войти по СМС.
2. Полностью закрыть и снова открыть приложение.
3. Пользователь остаётся авторизованным; «Профиль» показывает данные без повторного СМС.

## Проверка OpenAPI и моков

Сверить с [contracts/openapi.yaml](./contracts/openapi.yaml):

| Эндпоинт | Ожидание мока |
|----------|----------------|
| `GET /profile/me` | name, phone, email, emailVerified, pushEnabled |
| `PATCH /profile/me` | обновление name |
| `POST /profile/phone/request` | retryAfterSeconds: 60 |
| `POST /profile/phone/verify` | code `123456`; phone обновлён |
| `POST /profile/email/request` | retryAfterSeconds: 60 |
| `POST /profile/email/verify` | code `123456`; emailVerified: true |
| `PATCH /profile/preferences` | pushEnabled |
| `POST /auth/logout` | 204, сессия сброшена |
